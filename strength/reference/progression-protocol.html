Progression Protocol

Everything needed to decide what the bar does next session. Taught in Lesson 01.

Print this and keep it in your gym bag. It fits on one page.

1 · Rate the set: RPE and RIR

RPE = 10 − RIR. Rate the last set of an exercise; that's the one that tells you the truth.

RPERIRWhat it feels likeUse it for
100No reps left; another good-form rep would fail.Almost never
91One more, maybe. Not two.Isolation work, occasionally
82Two clean reps left; speed still good.Main lifts — the cap
73Three left. Clearly submaximal.Last warm-up, deload weeks

2 · The decision rule

Every main lift is programmed as 4 sets × 4–6 reps, RPE cap 8. After the session, walk the rules in order and stop at the first one that fires. The order matters: it is what makes the rule unambiguous when two conditions are true at once.

This rule runs in week 2. Not week 1.

Week 1 of a block is calibration. It caps at RPE 7, and its job is to find the loads rather than to progress them — so you do not walk this rule in week one at all. Use the week-one ramp in the block plan instead: if a set came in under RPE 7, add 5 kg (lower body) or 2.5 kg (upper body) and go again on the next set. Whatever lands at RPE 7 for a set of five is your week-two starting load.

This is not a technicality. The rules below are written against a cap of 8. Run them on a week-one session and a set of six at RPE 8 fires rule 3 — add load — off a set that just blew that week's own cap of 7, which §7 says is not earned. The rule and the programme would be contradicting each other. They don't: the ordered rule is a week-two instrument.

The knee tripwire in §5 is the exception. It runs in both weeks, because it was never part of the ordered rule.

#If… (week 2)Then
1 Any set fell below 4 reps, or the last set hit RPE 10 Drop load, rebuild up
2 Same load, no rep added, 3 sessions running Drop load, rebuild up
3 Every set hit 6 reps, RPE ≤ 8 Add load, reset reps
4 Last set went above RPE 8 — that is 8.5, 9, or 9.5 Repeat load and reps
5 Otherwise — every set 4–6, RPE ≤ 8, not all at 6 Add reps, keep load
Why ordered, not matched

An earlier version of this table listed five independent conditions. It was broken. A session of 6,6,6,6 @ RPE 10 satisfied two rows at once, and 5,4,4,4 @ RPE 9 in week one satisfied none. Reading top-to-bottom and stopping at the first match makes the rule complete: every possible session lands on exactly one action.

Why rule 4 says "above 8" and not "9"

It used to say RPE 9, exactly. Then an 8.5 — an ordinary half-point rating — walked straight through the entire table. Not rule 1: it isn't a 10. Not rule 3: it's over the cap. Not rule 4: it isn't a 9. It fell into rule 5 and was rewarded with an extra rep, at an effort that had already gone past the cap. Exactly backwards.

"Above 8" is what rule 4 always meant: you went over the cap — hold, don't progress. Nothing moves for whole numbers — an RPE 9 lands on rule 4 exactly as it always did, so every answer you have already learned is still the right one. Half points are welcome. Quarter points are not: the scale is not that precise, and pretending it is would be false confidence of the kind this page exists to avoid.

Load increments

  • Upper body (bench, overhead press): +2.5 kg
  • Lower body (squat, deadlift): +5 kg
  • Rule 1 drop: −5–10% of the working load
  • Rule 2 stall reset: back to ~90% of the stalled load, then climb again

3 · The minimum viable log

Per main lift, three numbers. That is the entire input to the rule above:

  • Load — kg, plus which machine if it isn't a free bar.
  • Last-set reps.
  • Last-set RPE — half points allowed. 8.5 is a real answer; 8.25 is not.

On lower days, add your knee /10. That number does not feed the ordered rule — it feeds the monitoring rule in §5, which runs independently and can stop a session the ordered rule would happily have progressed.

And say which week of the block it was. That is not bookkeeping: it decides whether the rule runs at all. Week 1 calibrates and the rule stays in its box; week 2 executes and the rule runs. One word, and it is the difference between "add load" and "you overshot — come back down."

This is a derived simplification, not a lowered standard. The rule has not changed. What changed is how much of the session you have to write down in order to run it — because the last set turns out to carry everything the rule actually reads.

Why the last set is enough

#What the rule asksDoes the last set answer it?
1 Any set below 4 reps · last set RPE 10 Yes — given the assumption. If the last set is your lowest-rep set, then last set ≥ 4 means every set was ≥ 4. The RPE half was already last-set-only.
2 Same load, no rep added, 3 sessions running Yes — once "no rep added" is pinned to the last set. It needs a history, not a fourth number. See the pin below: this one is a definition, not a deduction.
3 Every set at 6 reps, RPE ≤ 8 Yes — given the assumption. The rep range stops at 6, so no set can go above it. If your lowest set hit 6, all four did.
4 Last set above RPE 8 Yes, trivially. It only ever asked about the last set.
5 Otherwise Yes, trivially. Rule 5 is the residual. If 1–4 didn't fire, it fires — you never check its conditions at all.

Rules 1 and 3 rest on one assumption: your last set is your worst set. Reps do not go up across sets at a fixed load. That is very nearly always true — fatigue accumulates, and you are stopping at an RPE cap rather than at failure — but it is an assumption, not a law, and it is the only thing holding this up.

The blind spot — say this out loud, don't bury it

If an earlier set was worse than your last — you ground out a 3 on set two, then rallied — the three-number log hides it, and the rule misreads the session in one of two directions. Either rule 1 is missed (a sub-4 set goes unseen, so the load doesn't drop when it should), or rule 3 fires when it shouldn't (last set 6, earlier sets 5: the real rule says add reps, a last-set log says add load).

The escape hatch costs one sentence. If any earlier set felt worse than your last, say so — then write all four for that lift and we'll run it properly. You don't have to catch it perfectly. You only have to say "actually" when something was off.

The pin on rule 2

"No rep added" was ambiguous: added to which set? It is now pinned. A stall is three consecutive sessions at the same load with no rep added to the last set. That is a choice, not a derivation — and it makes rule 2 fire slightly sooner than a count-any-set reading would, because a rep gained on set one no longer holds off the reset. The direction is deliberate: this course resets early rather than grinding late.

None of this is an empirical finding. It is arithmetic about the rule as written — which is exactly why it is allowed to be this confident, and exactly why the one assumption underneath it has to stay in view. Log all four sets whenever you want; it is strictly more information and always welcome. But three numbers is enough to decide what goes on the bar, and three numbers you actually write down beat sixteen you don't.

4 · Rest between sets

WhereRestWhy
Main lift2–3 minTrained lifters need >2 min to express strength
Secondary compound~2 minSplit the difference
Accessories / supersets60–90 sFits the session; extend if reps or load drop sharply
Don't over-read the 90 seconds

Singer 2024 found a small benefit to resting longer than 60 s and did not detect a clear added hypertrophy benefit beyond ~90 s. That is weaker than "90 seconds is enough." Use 60–90 s on accessories because it makes the session fit — and take longer whenever reps or load fall off a cliff between sets.

5 · Your knee: the monitoring rule

You have a knee history and no current pain. That means you don't need rehab — you need a tripwire, so a niggle never becomes a diagnosis. Judge every knee-loaded session against all three:

CheckGreen lightBack off
Pain during the set≤ 3/10, dull, no sharpness> 3/10, or sharp / catching
Pain next morningBack to your normal baselineWorse than baseline at 24 h
Swelling or warmthNoneAny

If any column on the right is true: reduce range of motion, reduce load, or swap the movement — in that order. Two bad sessions in a row on the same lift means stop guessing and see a physiotherapist. I am not a clinician, and this page is not medical advice.

Where the 3/10 comes from — and why it isn't the study's number

The pain-monitoring model is real and randomised: Silbernagel et al. (2007) showed athletes who kept training through monitored pain did as well as those who rested. But the trial permitted loading pain up to 5/10, not 3.

Two reasons your number is stricter. The trial studied Achilles tendons, which tolerate load well — applying its threshold to a knee is already an extrapolation. And your mission is twenty more years of lifting, not a fast return to sport. When you extrapolate, extrapolate conservatively. The 3 is mine, not the evidence's. Now you know, and you can disagree with it on purpose.

6 · How much of this is actually proven?

Keep these labels in mind. A protocol you trust for the wrong reason is fragile.

ClaimStatusSource
Training to momentary failure is not required for growth Peer-reviewed Refalo 2023 (R9) — but actively contested; see below
Counting hard sets is a defensible volume metric Peer-reviewed Baz-Valle 2021 (R12) — validated for ~6–20+ rep sets
Counting our 4–6-rep main-lift sets in the weekly audit Approximation Below Baz-Valle's validated rep band. Pragmatic, not validated.
>2 min rest maximises strength in trained lifters Peer-reviewed Grgic 2018 (R10)
No detected hypertrophy gain past ~90 s rest Peer-reviewed Singer 2024 (R11) — absence of evidence, not evidence of absence
Double progression (fill reps → add load) Coaching construct Feigenbaum / Barbell Medicine (R6). No RCT found.
Three identical sessions = stall; reset to 90% Coaching construct Reasoned practice. No RCT found.
The knee monitoring rule (≤3/10, 24 h, no swelling) Clinical heuristic Widely used in rehab practice. Not a substitute for assessment.
+2.5 kg / +5 kg increments Convention What the plates allow.
The last set alone is enough to run the ordered rule Logical derivation Not evidence — arithmetic on the rule (§3). True if the last set is your worst set. Nobody has tested that assumption on you.
The contested one

Proximity to failure is the most argued-over variable in training. Refalo (2023) found momentary failure is not superior; newer analyses suggest that, volume-equated, closer-to-failure sets grow modestly more. Both camps sit inside 1–3 RIR. Train there and you are defensible whichever way the evidence settles.

7 · Glossary

Hard set unit
A working set taken close enough to failure to drive adaptation — in practice, 1–3 RIR. Warm-ups don't count.
RIR — reps in reserve effort
How many more good-form reps you could have completed. The honest currency of effort.
RPE — rate of perceived exertion effort
The same scale, inverted: RPE = 10 − RIR. Lifters talk in RPE; the maths is easier in RIR.
Double progression rule
Advance reps to the top of a range at a fixed load, then add load and reset to the bottom of the range. Reps first, load second — never both at once.
RPE cap rule
The maximum effort permitted on a working set. Your main lifts are capped at RPE 8 in week 2, and at RPE 7 in week 1, when the job is to find the load rather than to move it. A set that exceeds the cap doesn't count as earned, even if the reps were there — which is precisely why the ordered rule, written against the cap of 8, must not be run on a week-1 session.
Calibration week block
Week 1 of a block. Cap RPE 7, no progression, no ordered rule — you ramp the load between sets until a set of five lands at RPE 7, and that load is where week 2 starts. The knee tripwire still applies; it always does.
Stall diagnosis
Three consecutive sessions at the same load with no rep added to the last set. Not a bad day — a signal to reset, not to push. (The "last set" wording is the pin from §3; it is what makes the stall readable from a three-number log.)

Reconstruct the ordered rule from memory before you look at it. If you can't, that's not failure — that's the retrieval attempt doing its job. Ask me and we'll drill it.
