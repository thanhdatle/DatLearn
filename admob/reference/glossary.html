Mediation glossary

The vocabulary of AdMob mediation, defined once so every lesson uses the same words. Prints cleanly to one or two pages (⌘/Ctrl-P).

Mediation
The system that decides which ad network is asked to fill each ad request, and in what order — so multiple networks compete instead of relying on one. The umbrella this whole topic sits under.
Ad source = network
A demand partner that can serve an ad into your slot (AdMob itself, Meta Audience Network, Liftoff, Unity Ads, etc.). Mediation ranks and calls ad sources.
Mediation group
A configured set of ad sources for a given ad format and targeting. The container where you decide who competes and how (R4).
Waterfall
Mediation that calls ad sources sequentially, ordered by the average eCPM you set — each source is tried only if the ones above it fail to fill (R1).
Bidding = real-time bidding
Mediation that calls all participating sources simultaneously in a live auction; the highest actual bid wins (R2). Prices per impression instead of by your estimate.
Hybrid setup
A mediation group using both: the bidding auction runs first, and its winner is placed into the waterfall alongside your manually-ranked sources (R1).
eCPM effective cost per mille
Estimated earnings per 1,000 impressions. In waterfall it's the number you set to order sources; overall it's the headline efficiency metric for a placement.
Fill rate
Share of ad requests that actually return an ad (impressions ÷ requests). Low fill = empty slots = lost revenue; a job mediation exists to raise.
Impression
One actual display of an ad to a user. The unit everything is priced and capped in.
Frequency cap
A limit on how many times ads (app-wide or per ad unit) are shown to one user in a time window — the main dial for protecting retention (R7).
Ad formats
Banner (persistent strip), interstitial (full-screen at breaks), rewarded (opt-in for a reward), app-open (on launch/return). Each has different placement rules (R8, R9).
Latency
Time between requesting an ad and showing it. Long waterfalls add latency step by step; bidding runs one parallel round. High latency is a retention risk, not just an annoyance.
